## PET-ПРОЕКТ КУРСА:
**"Personal AI Coding Assistant — IDE расширение на Cursor"**

Во время курса студенты разработают собственного AI-ассистента для разработки:
- .cursorrules файл с кастомными инструкциями для своего стиля кода
- Интеграция собственного LLM (Ollama) в Cursor
- Prompt library для быстрого решения типовых задач
- Workflow для гибридной разработки (ручной код + AI-сгенерированный)
- Рефакторинг и оптимизация существующего кода через AI

## СТРУКТУРА КУРСА "Vibe-Coding: AI-первый подход к разработке"

### МОДУЛЬ 1: Введение в Vibe-Coding (50 мин)
**Занятие 1.1: Что такое Vibe-Coding и философия AI-first?** (15 мин)
- Vibe-Coding: программирование "на ощущение" с AI помощником
- Отличие от traditional coding: intent → LLM генерирует → refinement
- Примеры: ChatGPT, Copilot, Claude для кода
- Изменение мышления: от "писать код" к "описывать решение"

**Занятие 1.2: История и эволюция AI coding tools** (15 мин)
- GitHub Copilot: first-generation AI coding
- Cursor IDE: second-generation (context-aware)
- Claude, ChatGPT: LLM для архитектуры
- Будущее: AI agents самостоятельно пишут и тестируют

**Занятие 1.3: Инструменты: Cursor vs Copilot vs ChatGPT vs Cline** (20 мин)
- **Cursor IDE**: лучший для вибе-кодинга (фокус на Cursor)
- **GitHub Copilot**: встроен в VS Code, дешевле
- **ChatGPT + Copilot**: комбо для архитектуры
- **Cline / Claude Dev**: AI-агент для больших задач
- Когда что использовать

**Занятие 4.1: Что такое .cursorrules?** (10 мин)

**Занятие 4.2: Писать правила для своего стиля кода** (20 мин)

# - Переменные snake_case

# - Используй async/await для I/O операций
# - Тесты обязательны для новых функций

### МОДУЛЬ 5: Гибридный workflow: ручной код + AI (65 мин.)
**Занятие 5.1: Когда использовать AI, когда писать вручную** (15 мин.)
- Используй AI: шаблоны, тесты, документация, простые функции
- Избегай AI: критическая логика, безопасность, архитектурные решения, сложные алгоритмы
- Баланс: 80% AI + 20% ручное = оптимально

**Занятие 5.2: Workflow разработки функции** (20 мин.)
- Шаг 1: Описать intent в Cursor ("Напиши функцию для...")
- Шаг 2: AI генерирует первый вариант
- Шаг 3: Ты редактируешь и уточняешь (ручное улучшение)
- Шаг 4: Просишь AI добавить tests, docstrings
- Шаг 5: Code review в своей голове (или peer review)

**Занятие 5.3: Рефакторинг существующего кода** (15 мин.)
- Выделил кусок кода → "Сделай это более Pythonic"
- Выделил функцию → "Оптимизируй для скорости, используя..."
- Весь файл → "Переструктурируй с использованием паттерна..."

**Занятие 5.4: Pair programming с AI** (15 мин.)
- Cursor как напарник, а не полная замена
- Когда AI впереди (генерирует), когда ты впереди (решаешь)
- Как быстро итерировать

### МОДУЛЬ 6: Интеграция собственного LLM (Ollama) (50 мин.)
**Занятие 6.1: Зачем использовать local LLM?** (10 мин.)
- Приватность: код остается на машине
- Стоимость: бесплатно
- Скорость: иногда быстрее облака
- Недостатки: качество хуже, требует ресурсов

**Занятие 6.2: Установка Ollama и выбор моделей** (15 мин.)
- Установка Ollama (Mac/Linux/Windows)
- Скачивание моделей: `ollama pull mistral`
- Лучшие модели для кода: mistral, neural-chat, codeup
- Оценка размера модели vs качество

**Занятие 6.3: Интеграция в Cursor** (15 мин.)
- Settings → Models → Custom API
- URL: http://localhost:11434/v1
- Выбор модели в Cursor
- Тестирование на примере

**Занятие 6.4: Когда Ollama, когда облако?** (10 мин.)
- Ollama: повседневная работа, простые задачи, privacy data
- GPT-4/Claude: архитектура, сложные задачи, когда нужно лучшее

**Занятие 9.1: Code review AI-сгенерированного кода** (15 мин.)
